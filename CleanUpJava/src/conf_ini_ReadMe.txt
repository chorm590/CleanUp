
属性释义：

1、 debugable
	调试日志控制开关。值为true时程序在运行过程中会有较详细的日志打印。

2、	data_code_range
	随机数据的数值最大值，一般以ASCII为参考依据[0,127]。默认取值255，数值范围即为[0,255)。建议使用默认值。
	
3、	single_file_size_unit
	要保存的文件的大小的最小单位值。这条属性是要与下一条属性配合使用才具有意义的。可选值为：
		a) 1b		最小单位值为1个字节
		b) 1k		最小单位值为1KB
		c) 1m		最小单位值为1MB
		d) 1g		最小单位值为1GB
		e) others	最小单位值为256KB
	本单位数值不区分大小写。
	
4、	size_of_single_file
	生成的无意义数据文件单文件大小。与上第3条配合使用。如上第3条填入值‘1k’，而本条属性填入值‘16’。则表示存储的单个文件的大小为 16KB 。
	
5、	amount_of_files
	要生成的随机数据文件的数量。值 -1 表示无限制生成，直至磁盘没有更多空间用于存储。 极度不建议在大容量磁盘中选择小值大小文件存储，因为这有可能在一个磁盘中生成几十上百万甚至上亿个文件。
	
6、	fn_prefix
	输出文件的名称前缀。默认为 clean
	
7、	fn_suffix
	输出文件的名称后缀。默认为 override 。与上第6条配合，最终输出文件形如 clean1.override, clean2.override 等。
	
8、	output
	输出文件的目录，即要填满的那个磁盘的路径。
	
9、	append_files_in_sequence
	输出文件时是追加到现有文件的末尾还是从新开始。如上一次存储文件时并未将该磁盘存满，存到的最末尾一个文件为 'clean2236.override' ，若该属性的值为true，则下一次运程程序写入文件时的起始文件名即为 'clean2237.override'。
	当然这里有一个前提条件就是 fn_prefix 与 fn_suffix 要相匹配。
